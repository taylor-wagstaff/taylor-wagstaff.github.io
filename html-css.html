Explain control flow and loops using an example process from everyday life, for example, 'waking up' or 'brushing your teeth' (but not those).

Control flow is the order in which your computer reads and processes code, from top to bottom in the document. It starts from the very first line and ends in the last line. When writing code, we have to be aware of control flow and consider not only the order in which things are read, however in moments we might have to tell the computer to read different parts at different times.

Loops are iteration statements or a piece of code which tells the computer to perform a loop over and over again. There are many different times of loops, however, the most common is the ‘For Loop’ and it keeps continuing until there is nothing left to run through or it is based on a defined condition.

Now taking these two rough definitions of Control Flow and Loops, if we apply them to an everyday activity like running we can begin to understand them more conceptually. The Control Flow is the order or sequence of movements the body makes to perform one unit of running, such as the brain telling the foot to raise forward and plant itself ahead and the other leg next producing a sequence. Other movements such as breath, opening up of the chest, movement of the arms etc which are all part of the sequence and which are read quickly and in succession. Now the For Loop is this unit of running played out over and over again, that is until the body can no longer handle the sequence. When this happens the condition of being tired stops the loop, otherwise the person could run for infinity.

Describe what the DOM is and an example of how you might interact with it.

The DOM stands for Document Object Model, as it is defined by the logical structure in which it is accessed by the computer. All of your HTML, CSS and JavaScript files are either stored in a local or global server and the order in which these files are linked and structured determine how it is meant to be read by the computer. With the DOM, coders can build and create documents, adding and removing elements, write code or delete elements etc. In javaScript, I might use the DOM to access elements with different document types. For example, I could use the code document.GetElementById to locate an element of my choosing and then use .innerHTML to access it within the document.

Explain the difference between accessing data from arrays and objects.

An object is used when we create a named object, like a character, book title, product item or whatever name we wish to use to represent a digital object or container of information. An object is generally given a value and a key assigned to it. Such as:

Example 'person' object

var person = {

name: 'Taylor',

age: 14,

};

An array however we use to create a collection of multiple things in a given variable. Arrays are especially helpful when we use collections of a similar type and group them together to make it easier to deal with in code. Within an array, we can access them by number in the order that they are written out in code. Arrays can contain many things like objects can as well, such as strings, numbers, booleans, objects, or even other arrays.

Here is a basic array:

var painterNames = ['claudeMonet', 'jacksonPollock, ‘andyWarhol', 'joanMitchell'];

The difference between accessing data from arrays and objects is that when we want to access data from an object we refer to it by its variables we have placed inside of it. With an array however, we access through its numerical placement in the array.

Explain what functions are and why they are helpful.

Functions are one of the many building blocks of javascript. Functions are similar to a set of instructions which are executed by the computer. Within the scope of the function, which is between the curly branches, we define what we would like the computer to perform for us. We can declare a new variable, including operators, loops, booleans or many other things we would like it to perform. We usually give the function a name that relates to its purpose in the overall script.

Here is a simple function:

function square(number) {

return number * number;

}

In this function we can see it has a name of square, number is inscribed to its parameter, and within its scope we ask it to perform a return on the calculation of the number multiplied by itself. Meaning is number = 2, if we call the function that result would equal 4. Functions are useful in that we can make functions a standalone entity and define them in any way we want. This is what makes javaScript unique as it allows us to be able to refer to functions by name later on in our code, making it more compact and easier to deal with.

When to use Margins, Padding and Borders:


In regular use margins are good for adjusting the spacing of an element in conjunction with another element, i.e. positioning elements in relation with each other. Padding is good for when you're adjusting the look of a particular element like changing the space or amount of pixels contained within the box or div. Borders are useful to separate elements from other elements or containers. They are also handy when you're arranging elements on a page to clearly see where everything is located.
